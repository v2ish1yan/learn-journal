## The Problem It Solves

You spend an hour chatting with AI, solve several technical problems, and learn a lot.
But the moment you close the conversation, all that knowledge scatters across chat logs,
never to be found again.

**learning-journal** fixes this:

- **Automatically scans** the entire conversation when context is about to be compressed
- Extracts all knowledge points, tool usage, and reasoning processes
- Generates structured Markdown learning notes in a teacher's tone
- Supports Obsidian-style bidirectional links for building a personal knowledge graph

## Key Features

| Feature | Description |
|---------|-------------|
| **Teacher's Tone** | Not cold AI summaries, but lessons with intuition, analogies, and opinions |
| **AI Reasoning Restored** | Records not just conclusions, but *how* AI reached them — what clues it saw, what directions it ruled out |
| **Complete Hands-on Details** | Tool calls, scripts, addresses, byte sequences — nothing left out |
| **Domain-based Organization** | Notes organized by knowledge domain, not by question — for easy retrieval and cross-referencing |
| **Obsidian Bidirectional Links** | Auto-scans existing notes and creates `[[filename]]` associations |
| **Context Compression Guard** | Executes *before* context is compressed, ensuring no knowledge is lost |

## Quick Start

### Installation

Install this project as a Claude Code skill:

```bash
# Clone to your skills directory
git clone https://github.com/v2ish1yan/learn-journey.git
```

Reference SKILL.md in your Claude Code configuration, or use it directly in a project.

### Usage

**Auto-trigger:** When the conversation context is about to be compressed, knowledge extraction and document generation execute automatically.

**Manual trigger:** Type in the conversation:

```
/learning-journal
```

## Workflow

```
Conversation in progress
    │
    ▼

    │
    ▼
Group by knowledge domain ─── Not by question, by domain
```

## Configuration

### Output Directory

Learning documents are saved to `E:\AI\SKILL\learn\` by default. Modify this in SKILL.md.

### File Naming Convention

```
{YYYY-MM-DD}_{topic-slug}.md
```

Example: `2026-05-10_reverse-engineering.md`

### Tag System

Each note auto-generates 1–10 tags for knowledge classification and cross-note linking.

## Document Quality Standards

Documents generated by learning-journal follow strict quality standards:

- **No empty words** — Don't write "X is important because X", just explain the principle
- **No half-code** — Mention a tool? Give the full call. Mention a script? Paste the complete code
- **Preserve anchors** — Addresses, byte sequences, function names are kept intact
- **Restore reasoning** — Key reasoning nodes marked with `> 💭 Reasoning Process`
- **Anticipate confusion** — Proactively point out pitfalls newcomers often stumble into
